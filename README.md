# Local AI Chat Stack

A self-hosted AI chat stack built on **LiteLLM** + **Open WebUI**, backed by **GitHub Copilot** and **Moonshot AI (Kimi)**. Run multiple frontier models locally through a single OpenAI-compatible proxy — no per-token billing beyond your existing subscriptions.

## Architecture

```
Browser
  └── Open WebUI  :3000
        └── LiteLLM Proxy  :4000
              ├── GitHub Copilot API  (claude, gpt, grok)
              ├── Moonshot AI API     (kimi)
              └── PostgreSQL  :5432   (sessions, spend, keys)
```

| Service | Image | Port |
|---|---|---|
| Open WebUI | `ghcr.io/open-webui/open-webui:main` | 3000 |
| LiteLLM | `ghcr.io/berriai/litellm:v1.83.1-nightly` | 4000 |
| PostgreSQL | `postgres:16-alpine` | — (internal) |

## Setup

### 1. Clone and configure environment

```bash
cp .env.example .env
```

Edit `.env` and fill in the three required values:

```env
POSTGRES_PASSWORD=<strong random password>
LITELLM_MASTER_KEY=sk-<strong random string>
GITHUB_COPILOT_TOKEN=<see below>
```

### 2. Get your GitHub Copilot token

The token must be generated by the Copilot OAuth device flow — a regular GitHub PAT will not work.

Run the following from your terminal (uses Python 3, built into macOS):

```bash
python3 -c "
import urllib.request, json, time
CLIENT_ID = 'Iv1.b507a08c87ecfe98'
req = urllib.request.Request('https://github.com/login/device/code',
    data=json.dumps({'client_id': CLIENT_ID, 'scope': 'read:user'}).encode(),
    headers={'Accept': 'application/json', 'Content-Type': 'application/json'})
resp = json.loads(urllib.request.urlopen(req).read())
print(f'Go to:  {resp[\"verification_uri\"]}')
print(f'Code:   {resp[\"user_code\"]}')
print('Waiting for authorization...')
for _ in range(24):
    time.sleep(5)
    r = json.loads(urllib.request.urlopen(urllib.request.Request(
        'https://github.com/login/oauth/access_token',
        data=json.dumps({'client_id': CLIENT_ID, 'device_code': resp['device_code'],
            'grant_type': 'urn:ietf:params:oauth:grant-type:device_code'}).encode(),
        headers={'Accept': 'application/json', 'Content-Type': 'application/json'})).read())
    if 'access_token' in r: print('Token:', r['access_token']); break
"
```

1. Visit the printed URL
2. Enter the printed code
3. Paste the resulting `ghu_...` token into `.env` as `GITHUB_COPILOT_TOKEN`

### 3. (Optional) Add Kimi models

Sign up at https://platform.moonshot.ai, create an API key, and add it to `.env`:

```env
MOONSHOT_API_KEY=<your key>
```

### 4. Start the stack

```bash
docker compose up -d
```

Open WebUI will be available at **http://localhost:3000** in ~30 seconds.

## Volumes & Persistence

| Volume | What it stores |
|---|---|
| `postgres_data` | LiteLLM spend tracking, API keys, teams |
| `openwebui_data` | Users, chat history, settings |
| `copilot_cache` | Copilot session token — avoids re-auth on restart |

The `GITHUB_COPILOT_TOKEN` is written to the `copilot_cache` volume on every container start. LiteLLM automatically exchanges it for a short-lived Copilot session token and caches it. The session token is refreshed every ~25 minutes.

---

## Token Refresh

GitHub Copilot `ghu_` tokens can expire. If LiteLLM shows a device auth prompt in its logs:

```
Please visit https://github.com/login/device and enter code XXXX-XXXX
```

Authorize the code, then save the fresh token back to `.env`:

```bash
NEW_TOKEN=$(docker compose exec litellm cat /root/.config/litellm/github_copilot/access-token | tr -d '[:space:]') \
  && sed -i '' "s|^GITHUB_COPILOT_TOKEN=.*|GITHUB_COPILOT_TOKEN=${NEW_TOKEN}|" .env
```

## Adding Models

All models are defined in `litellm-config.yaml`. Add a new entry under `model_list`:

```yaml
- model_name: my-model          # name shown in Open WebUI
  litellm_params:
    model: provider/model-id    # e.g. moonshot/kimi-k2.5
    api_key: os.environ/MY_API_KEY
```

Then restart LiteLLM:

```bash
docker compose restart litellm
```

To list all available GitHub Copilot model IDs for your account:

```bash
docker compose exec litellm python3 -c "
import json
from litellm.llms.github_copilot.authenticator import Authenticator
import urllib.request
a = Authenticator()
token = a.get_api_key()
api_base = a.get_api_base() or 'https://api.githubcopilot.com'
req = urllib.request.Request(f'{api_base}/models', headers={
    'Authorization': f'Bearer {token}', 'Content-Type': 'application/json',
    'editor-version': 'vscode/1.85.1', 'editor-plugin-version': 'copilot/1.155.0',
    'user-agent': 'GithubCopilot/1.155.0', 'Copilot-Integration-Id': 'vscode-chat'})
resp = json.loads(urllib.request.urlopen(req).read())
for m in resp.get('data', []): print(m.get('id'), '-', m.get('name',''))
"
```

## Useful Commands

```bash
# Start the stack
docker compose up -d

# Stop the stack
docker compose down

# Restart a single service
docker compose restart litellm
docker compose restart open-webui

# View logs
docker compose logs litellm -f
docker compose logs open-webui -f

# Check running containers
docker compose ps

# Test a model via the LiteLLM API
source .env
curl http://localhost:4000/v1/chat/completions \
  -H "Authorization: Bearer $LITELLM_MASTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"claude-sonnet-4.6","messages":[{"role":"user","content":"hello"}],"max_tokens":20}'

# List available models
source .env
curl http://localhost:4000/v1/models \
  -H "Authorization: Bearer $LITELLM_MASTER_KEY" | python3 -m json.tool
```

## LiteLLM Admin UI

LiteLLM ships with a built-in admin dashboard at **http://localhost:4000/ui**.

Log in with your `LITELLM_MASTER_KEY` to:
- Monitor spend and usage per model and API key
- Create virtual keys with rate limits and budgets
- Add or remove models without restarting
- View request logs

---

## Open WebUI Features

- **Arena Mode** — compare two models side-by-side on the same prompt and vote on the better response. Select "Arena Model" from the model picker.
- **Chat history** — persisted in the `openwebui_data` volume across restarts
- **User accounts** — supports multiple users with separate chat histories
